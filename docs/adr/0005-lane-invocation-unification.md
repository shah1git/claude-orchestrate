# ADR-0005 — Унификация вызова агент-лейнов: объявление-данные + тонкий исполнитель

- **Статус:** принято (решение по существу + развилки F1, F7); срезовый план —
  дорожная карта, каждый срез ратифицируется на своём шаге
- **Дата:** 2026-07-22
- **Решил:** владелец репозитория (сессия обсуждения 2026-07-21/22); проект
  подготовлен architect-лейном (Fable/xhigh), утверждения о текущем состоянии
  перепроверены лидом на `main`
- **Затрагивает:** `skill/orchestrate/config.yaml`
  (`cross_provider.lanes`, `availability.*`); `skill/orchestrate/tools/`
  (новый `run-lane`; расширение `validate_config.py`); прозу
  `references/cross-provider.md`, `SKILL.md`; мост `/opt/tools/agent-bridge`
  (снимается с пути лейнов и удаляется — см. F7)

## Контекст

Оркестратор раздаёт подзадачи работникам разных вендоров. Способ вызова у
каждого свой: Claude-работники — Agent tool внутри харнесса; Codex — MCP-сервер
ЛИБО мост `run-external-agent.mjs`; Gemini и `agy-opus` — напрямую `agy`; Grok и
Kimi — свои CLI. Мост поддерживает лишь двоих из пяти (Codex и Gemini), причём его
gemini-путь сломан в коде.

Разбор 2026-07-21 показал, что команды запуска живут В ПРОЗЕ (таблицы
`cross-provider.md`, комментарии конфига) и разошлись с действительностью: в двух
местах лежала форма `agy -p --model …`, где `-p` съедает следующий токен как
промпт — модель не выбиралась, промпт терялся, канал отвечал приветствием. Полдня
ушло на разбор последствий. Тем же вечером обнаружены ещё два системных дефекта:
артефакт грейдился по УСЕЧЁННОЙ печати, а не по файлу с диска (модель, видя обрыв,
пересказывает результат по памяти — правдоподобно и неверно; полигон: по печати
48/450, по файлу 450/450); и обёртка `gemini-mcp-tool` молча выбрасывала выбор
модели. Общий корень всех трёх — **знание о форме вызова живёт в прозе или спрятано
в чужом слое, поэтому объявленное и исполняемое расходятся молча.**

## Решение

**Ни «один бинарник для всех», ни «оставить как есть». Правильна третья форма:
объявление вызова — ДАННЫЕ в `config.yaml`; форма вызова (argv) — КОД одного тонкого
исполнителя `tools/run-lane`, едущего С РЕПОЗИТОРИЕМ; лид не собирает командную
строку никогда.**

«Один бинарник» имеет два чтения: свой клиент вместо вендорских CLI — запрещено
(карта вендорских политик, `cross-provider.md:47-64`); одна программа-исполнитель,
через которую идут все вызовы, — это и есть рекомендация, при условии тонкости и
управления данными.

Контракт (одна команда на все лейны):

```
run-lane --lane <имя> --config <config.yaml> --prompt-file <файл>
         --workdir <абс.путь INPUTS> --out <абс.путь артефакта>
         [--role <имя>] [--schema <файл>] [--timeout <с>] [--resume <id>]
```

Ответ — один конверт JSON:
`{lane, transport, ok, model_declared, model_observed, effort,
artifact:{path,present,bytes,sha256}, printed_text, printed_truncated,
schema_enforcement, durationMs, usage, sessionId,
sandbox:{profile,applied,violations}, command, evidence:{log_path,model_line},
error}`.

`ok:true` требует ОДНОВРЕМЕННО: нулевой код выхода, присутствующий артефакт и
совпадение `model_observed` с `model_declared`.

Три свойства, ради которых всё затевается:
1. форма вызова не может разойтись с объявлением — в прозе её больше нет (валидатор
   запрещает командные строки внешних CLI в `SKILL.md` и `references/*.md`);
2. артефакт снимается с диска по обязательному `--out`, печать — отдельным полем с
   пометкой усечения (правило v25 `config.yaml:638-651` становится исполняемым);
3. фактическая модель сверяется по свидетелю — журнал agy
   (`Propagating selected model override`), события codex `--json`, kimi
   `stream-json`; расхождение = `ok:false`. Класс отказа обёртки `gemini-mcp-tool`
   закрывается механически.

Явный блок `capabilities` в объявлении лейна (`supports_effort`, `supports_schema`,
`has_own_sandbox`, `artifact_channel`, `model_verification`) остаётся: запрос
неподдерживаемой возможности — ГРОМКИЙ отказ, а не тихое отбрасывание (оба
сегодняшних провала — провалы тихого сокрытия).

### Развилка F1 (где живёт код исполнителя) — РЕШЕНА владельцем

Код исполнителя живёт РЯДОМ СО СКИЛЛОМ, в `skill/orchestrate/tools/`, там же, где
уже неделю лежат `validate_config.py`, `telemetry_append.py`, `preflight-fable.sh`.
Инвариант «скилл — чистый Markdown без кода» (`cross-provider.md:16-19`) не
отменяется, а переформулируется:

> Скилл не везёт ЧУЖОЙ код (вендорские CLI; мост `/opt/tools/agent-bridge`) — их он
> вызывает по наличию. Но СВОИ тонкие инструменты (валидатор, писатель телеметрии,
> исполнитель) держит в `tools/`, и они едут с репозиторием. Именно это даёт
> переносимость — цель № 1.

### Развилка F7 (судьба моста) — РЕШЕНА владельцем: мост удалить

Codex-логика (приведение схемы к strict, разбор `usage`, `sessionId`/`resume`)
переносится в `tools/run-lane`; мост `/opt/tools/agent-bridge` снимается с пути
лейнов ПОЛНОСТЬЮ и удаляется. Решающий довод: проект существует ради переносимости,
а зависимость пути лейна от НЕ едущего с репозиторием моста ей прямо противоречит;
gemini-путь моста и так сломан (`run-external-agent.mjs:432-437, :513-514`), а по
grok/kimi/claude он не умеет ничего. **Физическое удаление моста необратимо и
затрагивает другой репозиторий — оно выполняется на срезе 4 ПОСЛЕ доказанного
паритета `run-lane` по Codex, с явным подтверждением владельца перед удалением, а
не в момент принятия этого ADR.** ADR-0001 моста остаётся цитируемым каноном по
разделению «внешние агенты — воркеры, не второй оркестратор».

## Что уже сделано (коммит `14ce9fe`, срез 0)

Коммит закрыл три дефекта и один из них — тем самым механизмом, на котором строится
срез 1:

| Дефект | Состояние | Место |
|---|---|---|
| independence_filter молча пропускал `grok-4.5`/`sol-xhigh` вне карт | ИСПРАВЛЕН | `config.yaml` provider_map/surface_map |
| «Валидатор проверяет полноту» — обещание без кода | ИСПОЛНЕНО: `map_completeness_violations` | `validate_config.py` |
| `defaults` в отменённом состоянии v24 | ИСПРАВЛЕН к v26 | `config.yaml` |
| шапка v24 ссылалась на несуществующий `routed:` | СХЛОПНУТА | `config.yaml` |

Оговорка к срезу 1: `14ce9fe` закрыл дыру ценой ЛЕГИТИМИЗАЦИИ разнобоя — в
`provider_map` теперь сознательно перечислены и `sol`, и `sol-xhigh`, и `grok-build`,
и `grok-4.5`. Карта стала местом, где алиасы РАЗМНОЖЕНЫ, а не РАЗРЕШЕНЫ. Срез 1 это
разрешает (одно каноническое имя + словарь алиасов; vendor/surface — поля лейна,
карты производные), надстраивая существующую `map_completeness_violations`, а не
дублируя её.

## Шов

| Слой | Что живёт | Едет с репо |
|---|---|---|
| `config.yaml` → `cross_provider.lanes` | СЕМАНТИКА: vendor, surface, transport, model, effort, sandbox, hardening, capabilities, roles, status | да |
| `tools/run-lane` (свой код + тесты) | СИНТАКСИС: argv, закрытый stdin, обязательный `--add-dir`, env ужесточения, таймауты, разбор журнала, снятие артефакта, конверт | **да — это и есть переносимость** |
| `/opt/tools/agent-bridge` | ЧУЖОЙ код (снимается с пути лейнов, F7) | нет |
| Харнесс Claude Code | Agent tool для 4 работников, worktree, телеметрия, гейты | — |
| Проза скилла | МЕТОД: когда лейн уместен, что в тикет, ограничения ролей | да |

Ограничения осей (`delivery: inline-material`, `never: [live-tool-run,
self-walk-the-tree]`, `requires_pregate_output_inline`) переезжают из ЛЕЙНОВ в РОЛИ:
это свойства оси, а не канала, и именно из-за привязки к лейну одна Flash существует
пятью «лейнами».

## Claude-работники

Agent tool остаётся дефолтом; `claude -p` — второй транспорт того же лейна, только
на не-Claude хосте. `claude --help` даёт `--agent/--agents/--model/--effort/-p/
--add-dir` — те же `agents/*.md` достижимы снаружи; при вложенном спавне стереть
`CLAUDECODE`/`CLAUDE_CODE_ENTRYPOINT`. На не-Claude хосте `claude -p` теряет
подагентный worktree, штатную телеметрию и наследование контекста — это ДЕГРАДАЦИЯ,
а не паритет, и называть её надо так. Портируемый пол сохраняется: без
кросс-провайдерной поверхности прогон завершается на Claude-работниках.

## План миграции срезами

Каждый срез самостоятельно ценен, обратим и оставляет систему работающей.

- **Срез 0 — ЧАСТИЧНО СДЕЛАН (`14ce9fe`).** Полнота карт + замок
  `map_completeness_violations`.
- **Срез 1 — одно имя на исполнителя (остаток).** Канонические имена + словарь
  алиасов; инверсия карт (vendor/surface — поля лейна); расширение
  `map_completeness_violations` третьим условием «резолвится в лейн/алиас»
  (переиспользуя `_chain_members`, `_CHAIN_TERMINALS`). Только конфиг + валидатор.
- **Срез 2 — слить двойников.** 5 gemini-* → `gemini-flash`; ограничения ролей — в
  роли; растворить `defaults`; сжать `promoted`/`pilots`. Конфиг + проза.
- **Срез 3 — `tools/run-lane` с ОДНИМ адаптером (`agy`).** Перевести `gemini-flash`
  и `agy-opus`; удалить формы вызова agy из прозы; включить правило валидатора «в
  прозе нет командных строк внешних CLI». «Артефакт с диска» и сверка модели по
  журналу становятся исполняемыми.
- **Срез 4 — адаптеры `codex` и `claude-print`.** Codex-логику перенести в `run-lane`
  (strict-схема, `usage`, `sessionId`, `--sandbox`, `--resume`); ПОСЛЕ доказанного
  паритета — вывести мост с пути лейнов и удалить его (F7, с подтверждением владельца
  перед необратимым удалением). `claude-print` даёт переносимость на не-Claude хост.
- **Срез 5 — `grok` и `kimi`.** Адаптеры сами применяют `grok_hardening`/
  `kimi_hardening` и исполняют fail-closed гейт песочницы как ПРОВЕРКУ, а не прозу.
- **Срез 6 — телеметрия из конверта.** `telemetry_append.py` принимает конверт
  (`model_observed`, `usage`, `durationMs`, sha256 артефакта). Ручной ввод чисел
  исчезает.

## Риски и что мы теряем

| Риск | Как заметить | Как откатиться |
|---|---|---|
| `run-lane` — единая точка отказа лейнов | `--detect` по транспортам; `ok:false`+`error` | Claude-работники через него не идут — пол не затронут; лейн возвращается одной строкой `transport:` |
| Порт codex-логики даёт регресс | паритетный тест ДО удаления моста | мост не трогается до паритета; откат = вернуть прежний путь |
| Инверсия карт снимает замок раньше времени | валидатор красный, если поле лейна отсутствует | инверсия и расширение валидатора — один срез |
| Сверка модели опирается на форматы вендорских журналов | `model_verification: unavailable` в конверте — громко | лейн объявляет `model_verification: none`, работает с пометкой |

**Что теряем честно.** Мгновенность правки формы вызова: сегодня лид правит команду
в прозе и запускает; после — правка требует изменения кода с тестами. Сознательный
обмен скорости на невозможность расхождения.

**Где унификация ВРЕДНА:** семантические ограничения осей нельзя утапливать в
транспорт (они про доверие, не про канал); Claude-работников нельзя переводить на
`claude -p` ради симметрии (чистая потеря без выигрыша); возможности лейнов реально
разные (Codex — strict-схема; Grok — `--json-schema`; agy — схемы нет) — конверт
ОБЯЗАН сообщать `schema_enforcement` (`enforced` против `prompt_asked`), а не
выдавать их за одно.

## Следствия

- Форма вызова перестаёт быть прозой — класс расхождения, стоивший полдня, закрыт
  механически.
- Скилл переносим целиком: клонировал репозиторий, поставил вендорские CLI, запустил
  `install.sh` — кросс-провайдерная половина работает.
- Счёт сущностей падает: 5 gemini-* → 1; минус `defaults`, минус два рукописных блока
  карт (становятся производными), минус два блока пилотов, минус обёртка, минус
  `composer-build`, минус мост с пути лейнов.
- При следующей содержательной правке `config.yaml` поднять `version:` 25 → 26
  (комментарии `# v26` уже в файле; хеш телеметрии расхождение ловит, но значение
  должно совпадать).
