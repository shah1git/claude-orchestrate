# scout t2 — извлечение и классификация тарифных констант

## OBJECTIVE
Составить полный список **всех** тарифных констант уровня модуля в файлах
`depot/tariff.py` и `depot/booking.py` и классифицировать каждую по её префиксу.

## CONTEXT
Тарифные величины в проекте объявлены как константы уровня модуля с говорящими
префиксами. Нужен их точный реестр с категорией — механическое извлечение по
явному правилу, без суждений.

## INPUTS
- Файлы `depot/tariff.py` и `depot/booking.py` в выданном рабочем пространстве.
- Рассматривай **только** присваивания уровня модуля (нулевой отступ, не внутри
  функций/классов), имя которых начинается с одного из префиксов: `RATE_`, `COEF_`,
  `CAP_`.

## OUTPUT
JSON-массив объектов, по одному на каждую подходящую константу, в порядке: сначала
`tariff.py`, затем `booking.py`; внутри файла — в порядке появления сверху вниз.
Каждый объект ровно с четырьмя полями:
```json
{"file": "tariff.py", "name": "RATE_THEODOLITE", "category": "RATE", "value": "Decimal(\"1500.00\")"}
```
- `file` — `tariff.py` или `booking.py`.
- `name` — полное имя константы.
- `category` — префикс без завершающего подчёркивания: ровно одно из `RATE`, `COEF`, `CAP`.
- `value` — правая часть присваивания как она записана в исходнике (например
  `Decimal("1500.00")` или `6`), без завершающих комментариев и пробелов.

Верни **только** JSON-массив. Сохрани его в файл `answer.json` в корне рабочего пространства.

## TOOLS
Чтение и поиск по файлам. Инструмент выбираешь сам.

## BOUNDARIES
- Считаются только присваивания вида `ИМЯ = значение` на уровне модуля, где сразу
  за именем идёт знак `=`. Присваивания с аннотацией типа между именем и `=`
  (например `RATE_BY_CLASS: dict[...] = {...}` — словари-агрегаторы) **НЕ считаются**:
  за именем следует `:`, а не `=`.
- Константы внутри функций, локальные переменные, поля классов — не считаются.
- Не изменяй файлы проекта, кроме создания `answer.json`.

## ESCAPE HATCH
Если файлы отсутствуют или структура не совпадает с описанием — верни
`NEEDS_CLARIFICATION: <что увидел>` и ничего больше.

## ACCEPTANCE
Список сверяется с эталонным ключом: precision ≥ 0.95 И recall ≥ 0.95 по совпадению
(file, name) с точным соответствием category и value.
