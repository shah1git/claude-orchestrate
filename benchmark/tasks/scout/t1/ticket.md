# scout t1 — инвентаризация точек возбуждения ошибки

## OBJECTIVE
Составить полный список **всех** мест в коде, где возбуждается исключение
`DepotError`, в пакете `depot/` предоставленного проекта.

## CONTEXT
Проект — учебная фикстура сервиса проката оборудования. Каждый отказ бизнес-правила
выражается одной строкой вида `raise DepotError("DEP-###", "текст")`. Нужен точный
реестр этих мест для последующей сверки — задача чисто механическая, суждений не
требует.

## INPUTS
- Каталог `depot/` в корне выданного рабочего пространства (файлы `*.py`).
- Рассматривай только файлы верхнего уровня пакета `depot/` (не `tests/`).

## OUTPUT
JSON-массив объектов, по одному на каждое место возбуждения `DepotError`, в порядке:
сначала по имени файла (алфавит), затем по номеру строки (возрастание). Каждый объект
ровно с четырьмя полями:
```json
{"file": "booking.py", "line": 78, "code": "DEP-016", "message": "kit of {len(serials)} instruments exceeds the maximum kit size of {CAP_MAX_KIT_SIZE}"}
```
- `file` — имя файла без пути (например `booking.py`).
- `line` — номер строки, где стоит `raise` (целое, нумерация с 1).
- `code` — строковый код-литерал, первый аргумент (например `"DEP-016"`).
- `message` — текст второго аргумента **как он записан в исходнике между кавычками**,
  без окружающих кавычек, без префикса `f`. Фигурные скобки подстановок сохраняй как есть.

Верни **только** JSON-массив как единственное содержимое своего итогового ответа,
без пояснений до или после. (Если у тебя есть доступ на запись — можешь
дополнительно сохранить его в `answer.json` в корне рабочего пространства, но
основной формат сдачи — сам ответ.)

## TOOLS
Чтение и поиск по файлам. Инструмент выбираешь сам; результат — список выше.

## BOUNDARIES
- Только `raise DepotError(...)`. Определение класса `DepotError`, его импорты,
  строки документации, комментарии — не считаются местами возбуждения.
- Не изменяй ни одного файла проекта, кроме создания `answer.json`.
- Каждое место возбуждения умещается в одну строку исходника.

## ESCAPE HATCH
Если структура проекта не совпадает с описанием (нет каталога `depot/`, `raise`
записан иначе, чем в примере) — не угадывай. Верни строку
`NEEDS_CLARIFICATION: <что ты увидел вместо ожидаемого>` и ничего больше.

## ACCEPTANCE
Список сверяется с эталонным ключом: precision ≥ 0.95 И recall ≥ 0.95 по совпадению
(file, code) с точным соответствием line, code, message.
