# architect t4 — диагностика: отчёт о выручке завышает сумму брони

## OBJECTIVE
В выданной копии проекта `depot` есть один подсаженный дефект. Определи его
**первопричину**: конкретный файл и область кода, механизм, из-за которого
возникает наблюдаемый симптом, и причинную цепочку от кода до симптома.
Возвращать исправление не нужно — только диагноз.

## CONTEXT
Домен — прокат геодезического оборудования (см. `README.md` в корне фикстуры).
Стоимость брони считает `tariff.compute_total`: по коэффициентам полевого/
камерального дня и выходных, с потолком цены для длинных броней. Отчётность
(`reports.py`) по устройству берёт цифры **строго из журнала** (`Ledger`), а не
из «живых» объектов брони.

## SYMPTOM
Создаётся одна-единственная бронь. Возвращённый объект брони содержит
корректную итоговую стоимость (`booking.total_cost`). Однако отчёт о выручке —
при том что в системе эта бронь единственная — показывает **бóльшую** сумму,
чем итог самой брони. Расхождение детерминированно воспроизводится на выданной
копии.

## HOW TO RUN THE REPRO
В корне выданной рабочей копии (там, где каталог `depot/`):

```
python3 repro.py
```

Скрипт создаёт одну бронь и печатает `booking.total_cost` и отчёт о выручке.
Строка `SYMPTOM:` показывает, что отчёт больше итога брони, хотя бронь одна.

## OUTPUT
Верни: (1) FILE — файл и область кода первопричины; (2) MECHANISM — 2-4
предложения, почему возникает симптом; (3) CAUSAL CHAIN — пошаговый путь от
дефектного кода до симптома. Правку не предлагай.

## TOOLS
Обычные инструменты чтения/поиска и запуска Python (`python3 repro.py`,
при желании `pytest`). Правки в код вносить не нужно.

## BOUNDARIES
- Файл, область и исправление не даны — их и надо найти.
- Не изменяй ни один файл выданной копии — read-only диагностика.
- Ровно один подсаженный дефект.
- Фикстура выдаётся как копия `fixtures/depot-rc4`.

## ESCAPE HATCH
Если `repro.py` не запускается как описано — не угадывай. Верни
`NEEDS_CLARIFICATION: <что увидел>` и ничего больше.

## ACCEPTANCE
Ответ сверяется с заранее записанным ключом (файл + область, механизм,
причинная цепочка). Полный зачёт — назван файл И механизм И цепочка.
