# architect t2 — диагностика: недосчёт по субботам в тарифе

## OBJECTIVE
В выданной копии проекта `depot` есть один подсаженный дефект. Определи его
**первопричину**: конкретную строку кода, механизм, из-за которого она
производит наблюдаемый симптом, и причинную цепочку от этой строки до
симптома. Возвращать исправление не нужно — только диагноз.

## CONTEXT
Домен — учёт и бронирование геодезического оборудования (см. `README.md`
в корне выданной фикстуры). Тарификация описана в докстринге
`depot/tariff.py`: каждый день брони стоит `base_rate * day_coefficient`,
а выходные дни **дополнительно** умножаются на `COEF_WEEKEND_EXTRA` —
«weekend days additionally get multiplied by COEF_WEEKEND_EXTRA (equipment
sitting idle over a weekend still costs the depot in insurance and storage,
so we do not discount it)». Правило заявлено как относящееся ко **всем**
выходным дням недели, без исключений.

## SYMPTOM
Для брони, охватывающей диапазон дат с субботой внутри, итоговая сумма,
которую возвращает `compute_total`, оказывается **ниже**, чем сумма,
получаемая при ручном применении задокументированного правила (с
надбавкой за выходные, применённой ко всем выходным дням диапазона).
Расхождение воспроизводится детерминированно на выданной копии фикстуры.

## HOW TO RUN THE REPRO
В корне выданной рабочей копии (там, где лежит каталог `depot/`) выполни:

```
python3 repro.py
```

Скрипт считает `compute_total` для трёхдневной брони, начинающейся в
субботу 2026-01-03 (суббота проверена явно), и параллельно вручную
считает ожидаемую сумму по задокументированному правилу (надбавка — на
оба выходных дня диапазона). Строка, начинающаяся с `SYMPTOM:`, печатает
оба числа и разницу между ними.

## OUTPUT
Return: (1) FILE — the single file and line of the root cause; (2) MECHANISM — 2-4 sentences on why this code produces the symptom; (3) CAUSAL CHAIN — the step-by-step path from the flawed code to the observed symptom. Do not propose a fix.

## TOOLS
Обычные инструменты чтения/поиска по файлам и запуска Python
(`python3 repro.py`, при желании — `pytest`). Правку кода вносить не нужно.

## BOUNDARIES
- Файл, где находится причина, и само исправление раскрывать не нужно —
  их и предстоит найти; они не даны в этом тикете.
- Не изменяй ни один файл выданной копии — задача read-only диагностика.
- Рассматривается ровно один подсаженный дефект.

## ESCAPE HATCH
Если `repro.py` не запускается как описано (нет каталога `depot/`,
скрипт падает не на демонстрации симптома, а на импорте/окружении) — не
угадывай причину дефекта по догадке. Верни строку
`NEEDS_CLARIFICATION: <что ты увидел вместо ожидаемого>` и ничего больше.

## ACCEPTANCE
Ответ сверяется с заранее записанным ключом (файл + строка, механизм,
причинная цепочка). Полный зачёт — названы файл И механизм И причинная
цепочка; полузачёт — названо только место без механизма.
