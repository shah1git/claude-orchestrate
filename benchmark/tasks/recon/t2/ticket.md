# recon t2 — инвентаризация мест возбуждения ReconError (крупный корпус)

## OBJECTIVE
Составить полный список **всех** мест в коде, где возбуждается исключение
`ReconError`, в каталоге `corpus/` предоставленного рабочего пространства.

## CONTEXT
Корпус — синтетический набор Python-модулей. Каждый маркер — одна строка вида
`raise ReconError("RC-#####", "текст")`. Задача чисто механическая, суждений не
требует. **Масштаб этого тира:** ~200 файлов (~200K токенов) — давит скромные окна (haiku/sonnet ≈200K): здесь дешёвые тиры начинают ронять точность.

## INPUTS
- Каталог `corpus/` в корне рабочего пространства (файлы `*.py`).
- Рассматривай все модули верхнего уровня `corpus/`. Файл `errors.py` содержит
  только определение класса `ReconError` — это НЕ место возбуждения.

## OUTPUT
JSON-массив объектов, по одному на каждое место `raise ReconError`, в порядке:
сначала по имени файла (алфавит), затем по номеру строки (возрастание). Каждый
объект ровно с четырьмя полями:
```json
{"file": "mod_0007.py", "line": 12, "code": "RC-00042", "message": "kit operation 42 exceeds the configured kit_limit of {CAP_KIT}"}
```
- `file` — имя файла без пути.
- `line` — номер строки с `raise` (целое, с 1).
- `code` — литерал первого аргумента (например `"RC-00042"`).
- `message` — текст второго аргумента **как записан между кавычками**, без
  окружающих кавычек и без префикса `f`; фигурные скобки подстановок сохраняй.

Верни **только** JSON-массив как единственное содержимое итогового ответа. При
наличии доступа на запись можешь дополнительно сохранить `answer.json` в корне.

## TOOLS
Чтение и поиск по файлам; инструмент выбираешь сам. Как именно ты удержишь весь
корпус (в контексте или обходя инструментами) — часть испытуемого лейна.

## BOUNDARIES
- Только `raise ReconError(...)`. Определение класса в `errors.py`, импорты,
  строки документации, комментарии — не места возбуждения.
- Не изменяй ни одного файла корпуса, кроме создания `answer.json`.

## ESCAPE HATCH
Если структура не совпадает с описанием — не угадывай. Верни
`NEEDS_CLARIFICATION: <что увидел>` и ничего больше.

## ACCEPTANCE
Список сверяется с эталонным ключом: precision ≥ 0.95 И recall ≥ 0.95 по
совпадению (file, code) с точным соответствием line, code, message.
