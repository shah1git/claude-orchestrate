# builder t2 — рефакторинг с сохранением поведения: тарифный коэффициент дня

## OBJECTIVE
Вынести логику вычисления коэффициента одного дня брони (сочетание
коэффициента типа дня — полевой/камеральный — с множителем выходного дня)
из `compute_total` в новую публичную функцию `day_coefficient`, и заставить
`compute_total` вызывать её. Наблюдаемое поведение `compute_total` должно
остаться в точности прежним.

## CONTEXT
Сейчас в `depot/tariff.py`, внутри цикла по дням в `compute_total`, коэффициент
дня вычисляется инлайново:
```python
coef = COEF_FIELD_DAY if day_type is DayType.FIELD else COEF_OFFICE_DAY
if current_date.weekday() >= 5:
    coef = coef * COEF_WEEKEND_EXTRA
```
Эта логика (тип дня → базовый коэффициент, затем домножение на
`COEF_WEEKEND_EXTRA`, если день выходной) сама по себе не зависит от даты
как таковой — только от типа дня и признака «это выходной». Она заслуживает
быть отдельной, тестируемой функцией, а `compute_total` должен лишь решать,
какой день недели соответствует какому смещению, и передавать
день-тип/признак-выходного в неё.

Это рефакторинг, а не изменение поведения: ни одна константа, ни правило
округления (`ROUND_HALF_UP`, один раз, в конце `compute_total` — см.
докстринг модуля), ни логика потолка длинных броней (`CAP_TOTAL_*`,
`CAP_LONG_BOOKING_THRESHOLD_DAYS`) не меняются.

## INPUTS
- `depot/tariff.py` — единственный файл, который меняется.
- Прочитай его целиком, включая докстринг модуля (описывает модель
  ценообразования и правило округления) — рефакторинг обязан ему соответствовать
  дословно.

## OUTPUT
Новая публичная функция в `depot/tariff.py`:

```python
def day_coefficient(day_type: DayType, is_weekend: bool) -> Decimal:
    ...
```
- Возвращает `COEF_FIELD_DAY` или `COEF_OFFICE_DAY` в зависимости от `day_type`,
  домноженный на `COEF_WEEKEND_EXTRA`, если `is_weekend` истинно — ровно та же
  формула, что сейчас записана инлайново в цикле `compute_total`.
- `compute_total` вызывает `day_coefficient(day_type, is_weekend)` вместо
  инлайновой логики; вычисление `is_weekend` по дате (`current_date.weekday() >= 5`)
  остаётся в `compute_total`, так как оно требует конкретной даты дня, которой
  у `day_coefficient` нет.
- Всё остальное в `compute_total` (валидация, суммирование, применение потолка,
  округление один раз в конце) не меняется по существу — допустимы только
  правки, механически необходимые для вызова новой функции.
