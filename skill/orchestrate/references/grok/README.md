# Ужесточение Grok-лейнов

Основание — аудит открытого дерева `xai-org/grok-build` от 2026-07-18 (полный
отчёт файл:строка — в авто-памяти `grok-build-audit.md`). Три находки:

1. **Песочница по умолчанию `off`, встроенные профили fail-open.** `grok -p` без
   настройки работает без изоляции (объясняет инцидент 2026-07-15 «читает диск
   целиком»); даже профиль `workspace` читает весь `/`. Изолирует чтение только
   `strict`, но при неприменимости Landlock встроенные профили лишь пишут
   warning и продолжают **без песочницы**.
2. **Выгрузка в GCS xAI жива в коде, включается удалённо.** По умолчанию off, но
   remote settings сервера идут раньше дефолта. Пакует переписку, состояние
   сессии и изменения репозитория **включая untracked-файлы**; skip-лист
   исключает только сборочный мусор — `.ssh`, `*.pem`, `.env` НЕ исключены.
3. **Официальный бинарь несёт вшитые точки выгрузки** (эндпоинты/бакет/DSN на
   этапе сборки); в открытом дереве они пусты → самосборка структурно без них.

## Три уровня защиты (все в этом каталоге / в bootstrap)

- **`config.toml`** (здесь) → `~/.grok/config.toml`: выключатели телеметрии,
   выгрузки, Sentry, автообновления; профиль песочницы `orchestrate`. Локальные
   значения перебивают remote settings.
- **`~/.grok/sandbox.toml`** генерирует `bootstrap-mac.sh` с абсолютными
   deny-путями этой машины (тильда в deny НЕ раскрывается): кастомный профиль
   `orchestrate` наследует `strict` и запрещает `.ssh`/`.claude`/`.codex`.
   Наличие `deny` делает bubblewrap обязательным → **fail-closed** вместо
   fail-open встроенных профилей.
- **env спавнера лейна** (дублируют config, переживают подмену файла):
  `GROK_SANDBOX=orchestrate GROK_TELEMETRY_ENABLED=0
  GROK_TELEMETRY_TRACE_UPLOAD=0 DISABLE_ERROR_REPORTING=1
  GROK_DISABLE_AUTOUPDATER=1`

## Проверка в раннере (fail-closed)

После старта лейна: `~/.grok/sandbox-events.jsonl` должен содержать
`ProfileApplied`; строка `ApplyFailed` или «sandbox could not be applied» в
stderr — валить прогон, не доверять выводу.

## Пин версии

Цель — **самосборка 0.2.102** из открытого дерева (единственный бинарь без
вшитых точек выгрузки): `rust-toolchain 1.92.0`, `cargo build -p
xai-grok-pager-bin --release`. До готовности сборки лейны работают на
официальном бинаре, ужатом тремя уровнями выше (наружу — только инференс).
Наш 0.2.101 (07-14, до открытия кода) — вывести после пина самосборки.

## Важно

`strict`/`orchestrate` режет сеть дочерних процессов (seccomp): `npm/cargo
fetch` внутри grok упадут. Если билд-лейну нужна сеть — отдельный кастомный
профиль с `restrict_network = false` осознанным решением, не по умолчанию.
